# Git Workflow Rules

- Read current progress and recent changes before starting substantial work.
- Use `changelogs/changes/<slug>/` for meaningful features, refactors, or debugging work that spans more than a trivial edit.
- Keep worktree hygiene: do not revert unrelated user changes.
- Keep documentation, tests, and implementation aligned in the same logical change.
- Do not commit, amend, or push unless explicitly requested.
