# 07 - AI Agent Integration (NL to SQL)

> **Priority:** Medium | **Complexity:** High
> **Estimated Time:** 1-2 weeks

## Summary | Tóm tắt

**EN:** Integrate Ubiquitous AI Agents across all 7 Core Modules. While its primary capability includes Natural Language to SQL (NL-to-SQL) in the SQL Editor, it also features context-aware capabilities like suggesting schemas in Table Editor, optimizing indexes in Database Management, and analyzing stacktraces in Logs & Alerts.

**VI:** Tích hợp AI Agent toàn năng (Ubiquitous) phủ khắp 7 Core Modules. Ngoài khả năng dịch Ngôn ngữ tự nhiên sang SQL (NL-to-SQL) tại SQL Editor, AI còn tự động nhận diện ngữ cảnh (context-aware) để: gợi ý định dạng bảng ở Table Editor, phân tích lỗi ở Logs, và gợi ý phân quyền RLS ở cấu hình Database.

---

## Proposed Changes | Các thay đổi đề xuất

### 7.1 NL Interface Controller

**File:** `src/presentation/portal/data-builder/controllers/ai-agent.controller.ts`

**EN:**

- Create an endpoint `POST /api/v1/data-builder/ai/prompt` wrapped by presentation validation standard DTO pipes.
- Protect endpoint with standard auth guards to identify the calling user.

**VI:**

- Thiết lập endpoint `POST /api/v1/data-builder/ai/prompt` đi kèm DTO pipes để validation.
- Bảo vệ endpoint này với auth guard có sẵn để nhận diện người dùng.

### 7.2 AI Agent Use Case

**Files:**

- `src/application/data-builder/use-cases/ai/execute-ai-prompt.use-case.ts`
- `src/application/data-builder/services/ai-system-prompt.service.ts`

**EN:**

- Extend `BaseCommand` to handle `AiAgentCommand` carrying a `moduleContext` (e.g., 'SQLEditor', 'TableEditor').
- `AiSystemPromptService` injects context-specific instructions based on the active module (e.g., appending DB schema only when in SQL Editor context).
- Reaches out to the configured LLM API using an `Infrastructure` Port.
- Enforces Guardrails: Strips potentially destructive outputs or limits SQL generation strictly to non-destructive actions (`DROP`, `DELETE` blocked) before passing back to user.

**VI:**

- Kế thừa `BaseCommand`.
- Inject Service đọc Schema để nắm được các giới hạn của bảng trong whitelist.
- Gửi Prompt Context qua LLM API của hệ thống (sử dụng Port liên kết ngoài từ Infrastructure).
- Ráp nối Schema vào Prompt khéo léo để hạn chế model chế từ (hallucination).
- Chặn khẩn cấp các lệnh phá hoại như `DROP`, `DELETE`, `UPDATE` nếu AI vô tình sinh ra.

### 7.3 Integration with Runtime Engine

**EN:**

- Forward the cleaned query generated by the AI to the Runtime Engine `ExecuteViewUseCase` so the data translates reliably via the existing format.

**VI:**

- Truyền câu sql an toàn tạo được vào lại `ExecuteViewUseCase` đã lập trình ở module trước để hiển thị / trả về đúng định dạng table cho người dùng.

## Verification | Xác minh

- Users requesting "Delete all customers" should have their query firmly rejected by internal guardrails before querying the DB.
- Query bounds should be accurately scoped (e.g., "Show me my clients").
