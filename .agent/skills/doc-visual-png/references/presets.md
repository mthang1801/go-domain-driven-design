# Presets

Use `content_type` to choose the visual preset.

## `tech`

Use for engineering docs, code concepts, system behavior, APIs, runtime flow.

Style:

- cool palette
- precise contrast
- clean instructional tone

Good fit:

- `assets/go/fundamental`
- `assets/dsa`
- architecture or systems docs

## `glossary`

Use for concept definition, compare/contrast, term explanation, taxonomy.

Style:

- calmer palette
- softer card boundaries
- reference-friendly tone

Good fit:

- `assets/glosaries`
- concept-heavy docs with term comparisons

## `product`

Use for capability maps, planning docs, prioritization, feature framing.

Style:

- warm but structured
- product-map and flow-friendly

Good fit:

- product process docs
- planning or roadmap cards

## `game`

Use for gameplay loops, system maps, economy or progression comparisons.

Style:

- stronger contrast
- more energetic accents
- still readable in documentation

Good fit:

- game design docs
- gameplay system explainers

## `career`

Use for growth maps, communication frameworks, interview heuristics, learning paths.

Style:

- editorial tone
- slightly softer hierarchy
- reflective rather than mechanical

Good fit:

- career notes
- skill ladders
- coaching or self-review frameworks
